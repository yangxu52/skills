# Trigger Examples

Use these examples to sanity-check whether ` light-do ` should be invoked.

## Should Trigger

- "Use light-do for this small feature. I want a quick structured pass before coding."
- "This refactor touches a few files. Give it a lighter workflow than do."
- "Implement this clear feature with a short clarify-plan-implement-review flow."
- "The request is a bit fuzzy, but the change should be simple. Use light-do and clarify first."
- "I do not need tracked task artifacts, but I do want a disciplined workflow."
- "Use light-do to add this endpoint and wire the frontend call."

## Should Not Trigger

- "Fix this typo."
- "Rename this prop and update one test."
- "Brainstorm three possible architectures for this subsystem."
- "Use do for this risky migration with persistent artifacts."
- "This will probably span several sessions and needs tracked phases."
