# Trigger Examples

Use these examples to sanity-check whether ` do ` should be invoked.

## Should Trigger

- "Use do to refactor this authentication flow across backend and frontend."
- "Run this migration as a tracked task with phases and review."
- "I want a structured workflow for this risky multi-file change."
- "Use do for this feature. I want artifacts under .local/tasks."
- "This refactor is large enough that I want an explicit plan before implementation."
- "Use do to resume task dbadd1."
- "Use do to resume task 2026-04-17-login-refactor-a1b2c3."
- "Use do to reconnect task dbadd1."

## Should Not Trigger

- "Fix this typo in one file."
- "Explain why this test is failing."
- "Give me ideas for a possible design."
- "Review this short snippet."
- "Rename this variable and update the test."
- "List files under .local/tasks."
