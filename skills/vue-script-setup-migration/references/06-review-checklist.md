# Review Checklist

## 迁移结果复查

- 普通页面和普通业务组件已经改成 `script setup`
- 不再残留 Options API 的 `data`、`computed`、`methods`、`watch`、生命周期对象钩子
- 不再残留非特殊场景的非 `script setup` Composition API 业务壳
- 不再依赖 `this`、`mixins` 或标题文案驱动业务分支
- 不再把整页或整块业务状态重新糊进一个 `reactive()` 大对象
- 页面已经按承载面收敛，不再把多个独立流程硬塞进一个大组件
- 独立弹窗、独立提交流程、直接行操作已经被区分开
- 单页私有逻辑仍然留在所属 SFC，未为了整洁感默认外流成没有复用价值的 `helpers.js`
- 子组件依赖的组合式函数能力已在使用处按需获取，没有重新变成父层聚合后 props 透传
- 加载状态、提交状态、记录装载状态已经按职责拆开
- 旧式链式异步和不稳定的 loading 收口已经改成可维护的显式流程
- 表单初始值、校验器和提交流程不再依赖旧实例上下文
- 不再为了历史写法保留 `defineComponent`、`render()` 或 `h()` 外壳
- 如果保留了 `setup()` / `h()` / `render()`，已经能清楚说明它属于特殊场景，而且实现仍然是纯 Composition API
- 如果目标最终是 CRUD 列表页 + 弹窗伪页面，已经继续按对应 CRUD 规范做过收敛
- 无关旧代码、死状态、废弃辅助函数已经删除
